# Introduction
knowledge graph 可以很好地用来描述 relationship. <br>
knowledge graph 是一个**data structure** that stores information in `nodes` & `relationships`. 

Regular RAG 变成 → retrieve 1 chunk, traverse the graph to find other revevant chunks, giving LLM more **complete** context
- 普通RAG的 context depends on 这个chunk 有没有被召回到 top k- 如果语义差距较大，是有可能漏的

<br>

Node & relationships can both have `properties` (key value pairs)
- Nodes can be given `labels` to group them together.<br>
- Relationships always have a `type` and a `direction`.<br>
